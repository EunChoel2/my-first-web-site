React Native를 SDK보다 먼저 손대야 하는 이유

1. 앱의 기본 구조와 UI를 먼저 구현해야 함

앱 개발에서 가장 먼저 해야 할 일은 전체적인 구조와 UI(User Interface)를 만드는 것입니다.
QR 코드 스캔 기능을 구현하기 전에, 앱이 어떻게 작동할지, 사용자가 어떤 화면을 볼지, 각 기능들이 어떻게 연결될지를 먼저 정의해야 합니다.
예를 들어, 송장 QR 코드가 어떻게 표시될지, 카운트는 어디에 표시될지, QR 코드 스캔 버튼은 어디에 위치할지 등을 먼저 설계해야 합니다.
React Native는 이 과정을 쉽게 만들어주는 프레임워크이기 때문에, 먼저 React Native로 앱의 기본 뼈대를 잡아야 합니다.

2. 기능 구현 전에 상태 관리 및 로직 설계


앱에서 QR 코드를 스캔할 때, 스캔된 데이터를 어떻게 처리할지, 그리고 카운트를 어떻게 관리할지를 먼저 생각해야 합니다.
React Native에서는 이를 위해 useState, useEffect 같은 상태 관리 도구를 사용할 수 있습니다.
예를 들어, 제품을 스캔할 때마다 카운트가 1씩 증가하는 로직을 React Native로 구현하고, 스캔된 데이터를 표시하는 UI를 먼저 만들어야 합니다.

3. SDK는 필요한 기능을 나중에 추가

앱의 전체적인 구조와 UI가 잡히면, SDK를 추가해서 특정 기능을 구현하는 것이 다음 단계입니다.
예를 들어, QR 코드 스캔 기능을 추가하려면 react-native-qrcode-scanner 같은 SDK를 설치하고, 앱의 스캔 로직과 결합하면 됩니다.
하지만 이 과정은 기본적인 앱 구조가 먼저 갖춰진 후에 진행해야 효과적으로 통합할 수 있습니다.
SDK는 특정 기능을 쉽게 구현해주지만, 앱이 어떻게 동작할지 기본 구조가 없으면 그 기능을 통합하기 어렵기 때문에 나중에 추가하는 것이 좋습니다.
